v0327-db 六个检索接口对外文档

版本范围：v0327-db | 文档语言：中文 | 面向对象：下游系统、接口调用方、数据消费方

文档目标
本文档只覆盖 6 个对外检索接口： /memory/faces、 /memory/events、 /memory/vlm、 /memory/profiles、 /memory/relationships、 /memory/bundle。
文档中的样例以当前 v0327-db 实际返回结构为基准，字段名、层级和筛选规则与当前实现保持一致。

1. 统一约定

1.1 路由前缀

所有接口都挂在：

GET /api/users/{user_id}/memory/*

1.2 统一筛选参数

参数 说明
task_id最高优先级。指定后精确命中单个任务。
dataset_id指定某一组照片集合。
allfalse 只返回最近一条，true 返回当前筛选范围内全部任务。
scopedataset 只在当前数据集内查；user 跨用户全部数据集。
pipeline_version数字版 pipeline 版本，例如 327。
pipeline_channel版本通道，例如 db。
face_version / vlm_version / lp1_version / lp2_version / lp3_version / judge_version各阶段的数字版版本过滤。
updated_after按更新时间做增量拉取。
include_raw是否带回原始阶段数据。默认 false。
include_artifacts是否带回 artifacts。默认 false。
include_traces是否带回 agent traces。默认 false。

1.3 统一选择规则

  • 如果提供 task_id，接口直接返回该任务。
  • 如果没有 task_id，但提供了 dataset_id，就在该数据集内检索。
  • 如果两者都不提供，默认取最近一个数据集的最近一条任务。
  • all=false 返回当前筛选条件下最近一条任务。
  • all=true 返回当前筛选条件下的全部任务。
  • 所有版本字段都按整数精确匹配。

  • 凡是返回照片对象，都会给出稳定可展示 URL。
  • 凡是返回人脸对象，都会给出裁图 URL。

8. 对外接入建议

  • 如果下游只需要某一层能力，优先调用单阶段接口，不建议默认直接拉 bundle。
  • 如果下游需要做页面展示，所有照片都应优先使用返回的稳定 URL，而不是自行拼接路径。
  • 如果下游需要做审计、回放或模型复盘，可在支持的接口上开启 include_raw=true。
  • 如果下游需要增量同步，建议结合 updated_after 和 all=true 使用。
