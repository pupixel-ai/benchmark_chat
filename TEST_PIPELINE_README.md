# 测试脚本：test_pipeline.py

## 功能说明

随机从 `data/raw` 目录选择 N 张照片，运行主流程的前三步（加载 → HEIC转JPEG → 人脸识别），用于快速验证人脸识别效果，**无需等待 VLM 分析和事件提取**。

## 快速开始

### 基本用法
```bash
# 运行默认 10 张照片
python3 test_pipeline.py

# 运行 20 张照片
python3 test_pipeline.py --samples 20

# 运行 5 张照片，使用固定随机种子（可重现）
python3 test_pipeline.py --samples 5 --seed 42

# 指定自定义数据目录
python3 test_pipeline.py --samples 10 --data-dir /path/to/photos
```

### 命令行参数

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `--samples` | 10 | 采样照片数 |
| `--data-dir` | `data/raw` | 照片目录 |
| `--seed` | 无 | 随机种子（用于可重现结果） |

## 输出内容

脚本分四个步骤运行，并输出详细的进度信息：

### [1/4] 加载照片
- 扫描目录中的所有照片
- 读取 EXIF 信息（时间戳、GPS 位置）
- 按时间戳排序

### [2/4] 随机采样
- 从所有照片中随机选择 N 张
- 显示每张照片的：
  - 文件名
  - 拍摄时间
  - 位置（从 GPS 信息反向地理编码）

### [3/4] HEIC 转 JPEG
- 将 HEIC/HEIF 格式转换为全尺寸 JPEG
- 转换后的图片用于人脸识别

### [4/4] 人脸识别
- 使用 DeepFace (Facenet512 模型) 检测人脸
- 对每个检测到的人脸提取特征向量
- 与已有人物库匹配或创建新人物

## 结果摘要

脚本完成后输出以下统计信息：

### 📊 照片统计
- 总数、有人脸数、无人脸数

### 👤 人脸统计
- 检测到的人脸总数
- 识别出的不同人物数

### 📋 识别出的人物
对每个人物显示：
- 出现次数
- 首次和最后看到的时间
- 平均置信度

## 缓存与持久化

脚本写入标准缓存路径（与主流程一致）：

| 文件/目录 | 说明 |
|-----------|------|
| `cache/face_db.json` | 人脸库数据库（累积）|
| `cache/jpeg_images/` | 转换后的全尺寸 JPEG 图片 |
| `cache/test_pipeline_result.json` | 本次测试的详细结果 |

## 与主流程的关系

✅ **兼容性**：测试脚本的缓存与 `main.py` 完全兼容
- 后续可以直接运行 `main.py` 进行完整流程处理
- 不需要重新运行前三步（人脸识别）
- 节省大量处理时间

## 故障排除

### 没有检测到人脸
- 检查照片质量和清晰度
- 确认 EXIF 信息正确读取
- 查看 `调试` 输出中的人脸大小信息

### 相似度过低
- 检查 `config.py` 中的 `FACE_THRESHOLD` 设置
- 人脸特征质量可能不足
- 尝试增加样本数量

### 缓存冲突
删除旧的人脸库重新开始：
```bash
rm cache/face_db.json
python3 test_pipeline.py
```

## 技术细节

- **模型**：DeepFace with Facenet512
- **最小人脸尺寸**：70x70 像素（可在 config.py 调整）
- **相似度阈值**：0.70（可在 config.py 调整）
- **特征向量**：128 维
